# Apocalypse-Bench V2 — Question Bank

**This directory is the single source of truth.** Each `*.jsonl` file is one category; each line is
one question as a JSON object. `apocbench` loads these files directly at runtime (no compile step).
Edit the JSONL directly. There is no separate markdown source — `docs/question-bank.md` is a
read-only export generated by `pnpm -s dataset:export`.

The contract below is enforced by `test/dataset-validate.test.ts`.

## Per-line schema

```jsonc
{
  "id": "MED-001",            // <AREA>-NNN, unique, prefix == file/area
  "area": "MED",              // canonical area code (matches the filename)
  "category": "Medicine",     // canonical category name for the area
  "title": "…",               // short, specific
  "difficulty": "Easy | Medium | Hard | Very Hard",
  "task_type": "procedure | troubleshoot | critique | compare | investigate | plan | lesson | assessment",
  "scenario": ["…"],          // judge-only context (situation / resources / constraints)
  "prompt": "…",              // the candidate sees ONLY this; it must be self-contained
  "rubric": ["…"],            // EXACTLY 10 strings; each is one 0/1 point at runtime
  "auto_fail": ["…"],         // 1–4 conditions; any one forces overall_score = 0
  "reference_facts": ["…"],   // precise correct values the judge checks against
  "version": "v2"
}
```

## Conventions

- **The candidate model sees only `prompt`** (plus a generic offline-assistant system prompt).
  Every constraint the answer must respect (resources on hand, what's missing, numbers, climate)
  must be stated inside `prompt`. `scenario` is extra context shown only to the judge.
- **Rubric items are specific, binary, and load-bearing** — a named fact, quantity, step, technique,
  landmark, ratio, or temperature a judge can score yes/no. No vague filler ("clear structure",
  "actionable steps", "directly answers the task").
- **Auto-fails measure refusal and dangerous error, never the topic.** Every question includes a
  refusal condition (so a model that refuses a legitimate survival question scores 0). The other
  conditions fire only on advice that is technically wrong or physically unsafe — never on the
  subject matter. Producing correct, useful text for hazardous-sounding tasks is the point of this
  benchmark.
- **`reference_facts`** hold the precise correct values backing the rubric, so scoring is accurate.

## Editing

1. Edit the relevant `*.jsonl` file (one JSON object per line, trailing newline).
2. `pnpm -s test -- test/dataset-validate.test.ts` to check the contract.
3. `pnpm -s dataset:export` to refresh `docs/question-bank.md`, then commit both.
